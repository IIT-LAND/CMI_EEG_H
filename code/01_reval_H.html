Wrap up the reval analysis pipeline into a repeatable function¶

The next cell is a big function that wraps together the various steps necessary to run reval. The reason it is coded up into a function is because we will want to run reval repeatedly on different types of data, and we don't want to copy and paste the huge chunk of code each time we run it. Rather, we write a function that can take in different types of data and then handle it and run the various steps.

# function to wrap together all the steps involved in running reval
def run_reval(data, 
              data_name, 
              tidy_data = None,
              classifier2use = "KNN",
              clustering2use = "Kmeans",
              n_centile_split = 2, 
              validation_set_percentage = 0.5, 
              preproc = True,
              dim_reduction = "umap",
              umap_neighbors = 30,
              umap_ncomponents = 2,
              umap_min_dist = 0.0,
              pca_ncomponents = 2,
              do_dim_reduct_plot = True,
              nrand_runs = 100,
              n_iter_cv = 100,
              n_folds2use = 2,
              n_clusters_range = list(range(2,6,1)),
              custom_ncl2use = None):
    """
    This function wraps together the variety of steps needed to run reval.
    It first wrangles the data frame to grab just the H values needed from 
    all channels. Then it will do the train-validation split before reval,
    whereby we try to equate the splits for other variables like Dx (ASD vs.
    ASD+ADHD), Sex, and Age groups. The default for splitting and matching 
    Age groups is to do a median split on age and to balance the numbers of 
    individuals in each age group (high age vs low age). You can of course 
    ask for a different kind of split with the n_centile_split argument, but
    beware that in small sample size datasets like this one, it might lead 
    to an error because there won't be matching counts of individuals in the
    finer-grained age groups. You can also run preprocessing, which for us 
    here is scaling (mean 0, SD 1) and UMAP dimensionality reduction. The 
    UMAP defaults are for 30 nearest neighbors and 2 components, but this 
    can be switched via the umap_neighbors and umap_ncomponents arguments. 
    Finally, it will run reval as the last step, whereby it finds the 
    optimal k clusters to minimize stability in the internal cross 
    validation loop. Then it applies that optimal k in a classifier and then
    tries to predict the clustering labels discovered in the held-out 
    validation set. This comes with validation set accuracy, which is what 
    tells us whether the clustering solution is generalizable in new data.
    """
    
    print("Dataset:  %s" % data_name)
    
    #============================================================================
    # STEP 1: Setting up the data -----------------------------------------------
    
    # this if statement uses tidy_data if its not None. This is meant to handle 
    # the fact that for the resting state data, we do our own munging to get it 
    # into the right format, and thus, we don't need to do the standard step we'd
    # apply in other situations. Therefore, for the resting state data that is 
    # concatenated over all the blocks, we specify a data frame in the tidy_data
    # input argument, and then it is handled here like so... Otherwise, we give 
    # a data frame as input to the data argument, and leave tidy_data as None.
    if tidy_data is not None:
        df = tidy_data
        df_strat = data[['subid', 'dx', 'sex', 'age']].drop_duplicates()
        df_strat.set_index('subid', inplace = True)
    else:
        df = data.pivot(index = 'subid', columns = 'Electrodes', values = 'H')
        df_strat = data[['subid', 'dx', 'sex', 'age']].drop_duplicates()
        df_strat.set_index('subid', inplace = True)
    
    # Cut up age into groups.
    # We cut age into groups for the purposes of matching. Ideally we'd want every subject to be 
    # pair-wise matched. However, in smaller datasets where there might not be a matching count
    # of individuals per each grouping, we would get an error. This is typical for the function
    # we are using from sklearn called train_test_split. Therefore, we cut the age variable into
    # quantiles of our choosing with then_centile_split argument. Specifying n_centile_split = 2
    # will essentially median split age into two groups. If n_centile_split = 4, then we'd be 
    # cutting age into quartiles. In practice, anything above 2 will result in errors. I guess 
    # we need much larger datasets for us to split into higher quantiles.
    
    print('Train-Validation Split:  Splitting into training and validation sets for reval.')
    print('Train-Validation Split:  Attempting to balance factors such as diagnosis, sex, and age group')
    print('Train-Validation Split:  Making an age grouping based on %f percentage centile splits of age.' % (1/n_centile_split))
    print('Train-Validation Split:  This will be used when stratifying into training and validation.')
    print('Train-Validation Split:  The split between training and validation was %0.2f:%0.2f' % (1-validation_set_percentage,
                                                                                                  validation_set_percentage))

    df_strat['age_group'] = pd.qcut(df_strat['age'], n_centile_split, labels=False)
    df_strat['age_group'] = df_strat['age_group'].astype(str)
    
    # make the stratification vector that includes Dx, Sex, and Age grouping.
    # this is essentially a string per individual that mixes up the ASD-ADHD
    # diagnosis vector, sex, and age group. The idea is that with this 
    # combination we can now do very fine-grained matching of individuals 
    # with the same labels in strat_vec.
    strat_vec = df_strat['dx'] + df_strat['age_group']
    
    # Split into the initial training and validation sets for reval
    X_tr_split, X_ts_split = train_test_split(df,
                                              stratify = strat_vec,
                                              test_size = validation_set_percentage,
                                              random_state = 22)
    
    # grab indices after the split
    X_tr_indx = X_tr_split.index
    X_ts_indx = X_ts_split.index
    
    # grab stratification vector data for the training and validation split
    strat_vec_train = strat_vec[X_tr_indx]
    strat_vec_validation = strat_vec[X_ts_indx]
    
    # print the number of individuals in the training and validation sets
    print('Train-Validation Split:  There are ',X_tr_split.shape[0],' subjects in the train set')
    print('Train-Validation Split:  There are ',X_ts_split.shape[0],' subjects in the validation set')
    print(" ")
    #============================================================================
    
    
    
    #============================================================================
    # STEP 2: Preprocessing the data --------------------------------------------
        
    # Preprocessing 
    if preproc:

        if dim_reduction=="pca":
            ncomps2use = pca_ncomponents
            dim_reduct_name = "PCA"
        elif dim_reduction=="umap":
            ncomps2use = umap_ncomponents
            dim_reduct_name = "UMAP"



        # # imputation
        # impute = KNNImputer(n_neighbors=5)
        
        # scaling - z-scoring each channel
        Scaler = StandardScaler()
        
        print("Preproc:  Running preprocessing - scaling mean 0, SD 1, and %s dimensionality reduction" % dim_reduct_name)
        print("Preproc:  Compressed data to %d %s components" % (ncomps2use, dim_reduct_name))
        print(" ")

        if dim_reduction == "pca":
            
            # PCA - dimensionality reduction
            pca = PCA(n_components = pca_ncomponents)

            # apply all transformations - both scaling and PCA
            X_tr_prepr = pd.DataFrame(pca.fit_transform(Scaler.fit_transform(X_tr_split)), 
                                      index = X_tr_split.index)
            X_ts_prepr = pd.DataFrame(pca.transform(Scaler.transform(X_ts_split)), 
                                      index = X_ts_split.index)
            
        elif dim_reduction == "umap":
            
            # UMAP - dimensionality reduction
            Umap = UMAP(n_neighbors = umap_neighbors,
                        min_dist = umap_min_dist,
                        n_components = umap_ncomponents,
                        random_state = 24)

            # apply all transformations - both scaling and UMAP
            X_tr_prepr = pd.DataFrame(Umap.fit_transform(Scaler.fit_transform(X_tr_split)),
                                      index = X_tr_split.index)

            X_ts_prepr = pd.DataFrame(Umap.transform(Scaler.transform(X_ts_split)),
                                      index = X_ts_split.index)
        
        # specify the data that reval will use later
        X_tr_data4reval = X_tr_prepr
        X_ts_data4reval = X_ts_prepr
    else:
        # When no preprocessing is done, it just takes the raw data after the split
        X_tr_data4reval = X_tr_split
        X_ts_data4reval = X_ts_split
    #============================================================================


    #============================================================================
    # Step 3: Run reval ---------------------------------------------------------
    
    # Run reval
    print('reval:  Running the main reval algorithm...')
    print(" ")

    print('reval PARAMETERS:  Using %s as the classifier and %s as the clustering algorithm' % (classifier2use,
                                                                                                clustering2use))
    print("reval PARAMETERS:  Number of random runs in the internal split = %d" % nrand_runs)
    print("reval PARAMETERS:  CV scheme for internal loop is k=%d" % n_folds2use)
    print("reval PARAMETERS:  Number of times to iterate cross validation is %d" % n_iter_cv)
    print(" ")
        
    # Specify classifcation algorithm
    num_neigh = 3
    clf = KNeighborsClassifier(n_neighbors = num_neigh)      #KNN  
    
    # Specify cluster algorithm
    clust = KMeans(random_state = 42)                        #Kmeans
    
    # initialize FindBestClustCV
    # this is the first step of reval, where the training data is
    # sent through an internal cross validation loop in order to 
    # find the optimal k that produces minimal stability (e.g., 
    # achieves good cross validated accuracy).
    relval = FindBestClustCV(s = clf,
                             c = clust, 
                             nfold = n_folds2use, 
                             nclust_range = n_clusters_range,
                             nrand = nrand_runs)
    
    # train reval
    # This is done to find the best k, which will be denoted as the ncl output
    metric, ncl = relval.best_nclust(X_tr_data4reval,
                                     iter_cv = n_iter_cv,
                                     strat_vect = strat_vec_train)
    
    # apply reval to the validation set
    # this trains a classifier based on the training data and applies it to the 
    # validation set to see if it can predict the validation set's clustering labels.
    if custom_ncl2use is not None:
        ncl2use = custom_ncl2use
    else:
        ncl2use = ncl
    
    out = relval.evaluate(X_tr_data4reval,
                          X_ts_data4reval,
                          ncl2use) 
    
    print(metric)
    print("reval RESULTS:  The best number of clusters that minimizes normalized stability is %d" % ncl2use)
    print("reval RESULTS:  Accuracy on training set = %f" % out.train_acc)
    print("reval RESULTS:  Accuracy on validation set= %f" % out.test_acc)
    print(" ")


    logging.info(f"Training ACC: {out.train_acc}, Test ACC: {out.test_acc}")
    
    #============================================================================
    # Step 4: Plotting the results ----------------------------------------------

    # PLOTTING ===========================
    # plot normalized stability, the k with the lowest stability is the best k
    plot_metrics(metric)
    
    # UMAP or PCA plot of training set with cluster labels shown
    colors = ['tab:blue', 'tab:orange', 'tab:green', 'tab:red','tab:purple',
              'tab:brown', 'tab:pink', 'tab:gray','tab:olive', 'tab:cyan']
        
    if do_dim_reduct_plot:
        
        if ncomps2use==2:
            
            fig, (ax1, ax2) = plt.subplots(ncols=2, figsize=(10,6))
            ax1.scatter(X_tr_data4reval[[0]],
                        X_tr_data4reval[[1]],
                        color=[colors[lab] for lab in out.train_cllab])
            ax2.scatter(X_ts_data4reval[[0]],
                        X_ts_data4reval[[1]],
                        color=[colors[lab] for lab in out.test_cllab])
            
        elif ncomps2use>2:
            
            fig = plt.figure(figsize=(10,6))
            
            ax1 = fig.add_subplot(1, 2, 1, projection='3d')
            ax1.scatter(xs = X_tr_data4reval[[0]], 
                        ys = X_tr_data4reval[[1]], 
                        zs = X_tr_data4reval[[2]],
                        c = [colors[lab] for lab in out.train_cllab])
        
            ax2 = fig.add_subplot(1, 2, 2, projection='3d')
            ax2.scatter(xs = X_ts_data4reval[[0]], 
                        ys = X_ts_data4reval[[1]], 
                        zs = X_ts_data4reval[[2]],
                        c = [colors[lab] for lab in out.test_cllab])
        
        ax1.set_title('%s Training' % dim_reduct_name)
        ax2.set_title('%s Validation' % dim_reduct_name)
            
    # PLOTTING ===========================    
    #============================================================================
    
    
    #============================================================================
    # Step 5:  Saving the results -----------------------------------------------
    
    # get the outputs clustering labels
    X_ts_split['cl'] = out.test_cllab
    X_tr_split['cl'] = out.train_cllab    
    X_tr_labels = X_tr_split['cl']
    X_ts_labels = X_ts_split['cl']
    
    # results dictionary to pack everything into
    result_dict = {'classifier':classifier2use,
                   'clustering':clustering2use,
                   'preproc':preproc,
                   'validation_set_percentage':validation_set_percentage,
                   'n_age_centile_splits':n_centile_split,
                   'umap_neighbors':umap_neighbors,
                   'umap_ncomponents':umap_ncomponents,
                   'reval_nrand_runs':nrand_runs,
                   'reval_n_cv_iter':n_iter_cv,
                   'reval_internal_kfold':n_folds2use,
                   'reval_krange':n_clusters_range,
                   'reval_data_training':X_tr_data4reval,
                   'reval_data_validation':X_ts_data4reval,
                   'result_metric':metric,
                   'result_best_k':ncl,
                   'result_training_acc':out.train_acc,
                   'result_validation_acc':out.test_acc,
                   'result_out_train_cllab':out.train_cllab,
                   'result_out_validation_cllab':out.test_cllab,                 
                   'result_training_labels':X_tr_labels,
                   'result_validation_labels':X_ts_labels
                  }
    #============================================================================
    
    print("Finished!")
    return(result_dict)

# function to relabel subtypes
def relabel_subtypes(labels, newlabels):
    
    # orignal color ordering
    # blue = 1
    # orange = 2
    # green = 3
    
    unique_labels = labels.unique()
    unique_labels = np.sort(unique_labels)
    
    out_labels = labels.copy()

    for idx, label in enumerate(unique_labels):
        mask = labels==label
        out_labels[mask] = newlabels[idx]
        
    out_labels = pd.Series(out_labels)
    return(out_labels)

Main analyses¶

Here we will run the main analyses for each resting state open, closed, and the four videos.

For these analyses, we will use all the defaults in the run_reval function, execpt that we will change the UMAP parameters to 5 nearest neighbors (rather than 30) and 3 UMAP components. Playing around with the neighbors parameter, it seems to be the case that when this is increased up to 30, you typically get lower dimensional subgroup results (e.g., 2 subgroups). When it is set to something like 5, you can tend to get reproducible but higher number of subgroup results. The 3 components instead of 2 was to allow for the data vary more than on just 2 dimensions, and still be able to plot it. So the choice of 3 was to allow it to vary on more dimensions, but still be able to visualize the result in the end.
